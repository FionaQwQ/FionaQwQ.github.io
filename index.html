鉴权方式及相关概念

Posted on 2020-02-15 In 前端工程

核心概念

  • 鉴权≠算法/加密≠HTTPS

    鉴权

  • 基础鉴权
  • session/cookie
    优点：较易扩展；简单
    缺点：安全性低；性能低，服务端存储；多服务器同步session困难；跨平台困难
  • JWT（Json Web Token）
    优点：易扩展；支持移动设备；跨应用调用；安全；承载信息丰富；无状态，编码数据
    缺点：刷新与过期处理；Payload不易过大；中间人攻击
  • Oauth
    优点：开放；安全；简单；权限指定
    缺点：需要增加授权服务器；增加网络请求
  • …

算法/加密

  • Base64
  • MD5/SHA-1
  • DES/AES
  • RSA/ECC
  • …

HTTPS

  • SSL
  • HTTP劫持
  • 数据篡改
  • 敏感信息
  • 中间人

JWT工作原理

  • JSON WEB TOKEN
  • 一个JWT由三部分组成：头部(head)，载荷(payload)，签名(signature)
  • 一个好玩的jwt网站

VUE是什么

  • 构建用户界面的渐进式框架

VUE特点

  • 自底向上
  • 核心库只关注视图层
  • 易上手
  • 便于与第三方库或既有项目整合

兼容性

  • 不支持IE8及以下版本
  • 支持所有兼容ECMAScript 5的浏览器

安装

  • 直接使用<script>引入

Vue会被注册为一个全局变量

  • NPM

  • 命令行工具（CLI）

声明式渲染

Vue.js的核心是：
一个允许采用简洁的模板语法来声明式地将数据渲染进DOM的系统。
看起来与渲染一个字符串模板非常类似，但是Vue在背后做了大量工作。
数据与DOM之间建立关联，所有东西都是响应式的。

  • 文本插值

指令

  • v-bind
    指令绑定元素特性
    在渲染的DOM上应用特殊的响应式行为
  • v-if
    即不仅可以把数据绑定到文本或特性，还可以绑定到DOM结构
  • v-for
    v-for指令可以绑定数组的数据来渲染一个列表等
  • v-on
    可以添加事件监听器，通过它调用在Vue实例中定义的方法
  • v-model
    轻松实现表单输入和应用状态之间的双向绑定

BFC

Posted on 2019-12-27 In css

BFC定义

block formatting context，块级格式化上下文

BFC特性

BFC可以使一个元素像一个围城，里面的人出不去，外面的人进不来。
如果一个元素具有BFC，内部子元素无论如何都不会影响到外部的元素。
所以——

  • BFC不可能发生margin重叠
  • BFC可以用来清除浮动带来的影响（因为如果清除不了，子元素浮动使父元素高度塌陷，会影响其后元素布局与定位，显然有违BFC元素的子元素不会影响外部元素的设定）
    当然这些并不是BFC特性最终目标，BFC最重要的用途是实现更健壮、更智能的自适应布局。

BFC触发条件

  • <html>根元素
  • float的值不为none
  • overflow的值为auto\scroll\hidden
  • display的值为table-cell\table-caption\inline-block
  • position的值不为relative\static

只要符合其上所列任意一个条件，就无须使用clear: both属性去清除浮动的影响了

BFC与流体布局

.text的内容受到设置了float属性值的图片的影响而环绕图片，此时给.text设置具有BFC特性的属性，比如overflow: hidden。

由于具有BFC特性的元素的子元素不会受外部元素影响，也不会影响外部元素。所以这里.text元素为了不和浮动元素产生任何交集，顺着浮动边缘形成自己的封闭上下文。
即普通流体元素在设置了BFC特性属性后，会自动填满容器中除了浮动元素以外的剩余空间，形成自适应布局效果。我们让图片的尺寸变大或变小，右侧自适应内容无须更改任何样式代码，都可以自动填满剩余的空间。

如果想令图片和文字保持合适的间距，则该例中

  • 图片可以设置以下任意一条：
    img { margin-right: 10px; }
    img { border-right: 10px solid transparent; }
    img { padding-right: 10px; }

  • 或者.text可以设置以下任意一条：
    .text { boder-left: 10px solid transparent; }
    .text { padding-left: 10px; }

注意，不包括.text { margin-left: 10px; }，因为其值必须是浮动元素的宽度加间隙的大小，所以会变成动态不可控的，无法大规模复用

作用

  • 打包你的样式
  • 打包你的资源
  • 打包你的脚本
  • 资源压缩
  • 代码混淆
  • 代码检查
  • 转义
  • 预处理
  • 开发-热更新
  • ……

安装

  • 查看node版本

  • 切换node版本
    若需要切换node版本，可使用nvm
    nvm下载方式
    记得设置环境变量

    切换node版本方式：

  • 创建nodejs项目
    使用npm init -y快速创建nodejs项目

  • 局部安装webpack

  • D和–save-dev表示把它安装成开发模式下的依赖，不会去打包到正式打包的项目当中

  • 查看版本

  • 全局安装webpack

  • 查看版本
    就可以直接使用webpack命令查看版本

配置文件

webpack默认配置文件: webpack.config.js
可以通过webpack –config 指定配置文件

配置入口

告诉webpack我们的应用从哪个js开始，你从哪个js开始去阅读我们的代码，去查阅我们的代码里面有什么样的问题，能够从代码里面去做哪些关联性的操作，比如说压缩代码

配置输出

告诉webpack最终你打包出来的名字是什么，路径是什么

loader

帮我们处理一些静态资源文件，比如说让webpack拥有对一些高阶语法的支持（如ES6——babelloader）、比如说支持图片字体等资源（urlloader、fileloader）。
这些loader为webpack提供了扩展能力和处理资源的能力。

插件

插件支持loader无法实现的一些功能，webpack官方提供了很多很优秀的插件，从打包优化到压缩等等。
插件赋予了webpack无限扩展的能力

模式/兼容性

development 开发模式
production 生产模式
